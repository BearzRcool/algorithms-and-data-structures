Data Structures
-a way of storing and organizing (lists, dictionaries)
-Stacks
--old way to talk of lists
--linear
--lifo, last in first out
--can only access the first(top-most) thing in the stack
--Can do 5 things
---is Empty() - checks if it is Empty
---pop() - removes the last thing you put in (lifo)
---push() - adds thing to the top
---peek() - checks what is at the top
---size() - checks how many things are in the stack
--uses probability and randomness to make bots that can do complicated things
--used for saving memory
-Queues
--opposite of stacks, filo (first in last out)
--things it can do:
---enqueue() - adds something to the beginning of the queue (pronounced on-queue)
---dequeue() - removes from the beginning of the queue
---peek() - checks the beginning, not the end, unlike a stack
---isEmpty() - same as a stack
---size() - same as a stack
-Link lists(regular list)
--combination of queues and stacks
--3 types
---Singly Linklist
---list of things linked together, nodes
---only connected going forward, meaning each node keeps track of 2 things
---what it is and which node comes next
---[a] > [b] > [c]
--Doubly Linklist
---same as singly, but each node also knows what node is before it
--- [a] <> [b] <> [c]
--Doubly Circuly Linklist
---same as doubly, but the first node(head) knows about the last node(tail) and vice versa, making it a circle
